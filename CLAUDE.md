## Commands

```bash
npm start          # Local dev server (localhost:3000)
npm run build      # Production build (outputs to build/)
npm run serve      # Serve the production build locally
npm run clear      # Clear Docusaurus cache (run when builds behave unexpectedly)
```

There is no test suite. TypeScript type checking is the primary correctness tool.

## Architecture

This is a **Docusaurus v2** static site for the Robotics @APEX club at SST Singapore. It generates a deployable site from React pages, SCSS-styled components, and Markdown docs.

### Two content systems in one

1. **React pages** (`src/pages/`) — custom UI pages (home, achievements, projects, join, about). These are fully hand-crafted components with their own data loading.
2. **Docusaurus docs** (`docs/`) — Markdown/MDX documentation for each robotics tech stack (VEX, FLL, Arduino, etc.), with autogenerated sidebars defined in `sidebars.js`.

### Achievements data pipeline

The achievements page is the most data-intensive part of the site:

- Source of truth: `data/achievements/achievements.json`
- Structure: array of `{ year, competitions: [{ code, region, desc, awards: [{ title, category, team, recipients }] }] }`
- **Critical format rule**: awards for the same `code` + `region` within a year must be grouped into a single competition entry with multiple items in `awards[]`. Each competition code should appear at most once per region per year.
- Competition codes and their metadata (full names, region-specific names, websites) are defined in `src/types/competitions.ts`
- Achievement images are loaded from GitHub raw content URLs based on file paths under `data/achievements/[year]/[code]_[region]/`
- If an image is missing, it falls back to a placeholder from cataas.com

### Projects data

The `/projects` page fetches live from the GitHub API (`/orgs/roboapex/repos`) using SWR. Repos are filtered and parsed by naming convention: `[YEAR][4digits]-[event]-[team]-[other]`. The repos `roboapex.github.io` and `roboapex` are excluded.

### Styling

- Global theme variables and overrides: `src/styles/custom.scss` (Docusaurus CSS variable overrides go here)
- Component styles: SCSS modules co-located with components (e.g., `styles.module.scss`)
- Responsive breakpoint: 996px (matches Docusaurus default)

### TypeScript paths

`tsconfig.json` extends `@tsconfig/docusaurus` with no custom path aliases — imports use relative paths.

### Key config files

- `docusaurus.config.js` — navbar links, footer, site URL, plugins
- `sidebars.js` — autogenerated sidebar config per tech stack
- `baseDocPaths.js` — maps tech names to their `/docs/` subdirectory paths
- `constants.js` — social media links (Instagram, GitHub, email)

### Deployment

Pushing to `main` triggers `.github/workflows/main.yml`, which calls a custom deployment API endpoint (secrets: `DEPLOY_URL`, `DEPLOY_TOKEN`, `DEPLOY_APP_ID`) rather than using `docusaurus deploy` directly.
